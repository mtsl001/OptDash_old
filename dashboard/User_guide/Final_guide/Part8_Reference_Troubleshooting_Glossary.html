8.1

FastAPI Endpoint Reference

All endpoints are served at http://127.0.0.1:8000. The Swagger interactive docs are available at http://127.0.0.1:8000/docs. Query parameters are shown as ?param=value.

Method Endpoint Params Returns
GET /api/gate date, underlying Gate score (0–11), 8-condition details (6 core + 2 VEX/CEX bonus), verdict (GO/WAIT/NO-GO)
GET /api/gex/timeseries date, underlying Net GEX per snapshot, daily peak, GEX%
GET /api/coc/timeseries date, underlying CoC per snapshot, V_CoC 15-min velocity array
GET /api/screener date, underlying, option_type All ATM-window strikes ranked by S_score with all columns
GET /api/pcr/divergence date, underlying PCR_vol, PCR_oi, PCR_div per snapshot
GET /api/alerts date, underlying Chronological list of fired alerts with type, time, metadata
GET /api/iv/surface date, underlying, snapshot_time IV matrix: strikes × expiries, IVP per cell
GET /api/iv/term-structure date, underlying, strike ATM IV per expiry tier for a specific strike
GET /api/position/pnl entry_price, lots, strike, option_type, underlying, expiry, entry_time Live P&L, Greek decomposition, SL_t chart data
GET /api/market/spot underlying Latest spot price (from most recent snapshot)
GET /api/atm-window date, underlying ATM strike, ±8 strike array, strike interval used
GET /api/watermark — Last successful pipeline pull timestamp
GET /api/calendar/is-trading-day date {"is_trading": true/false, "reason": "..."}
GET /api/health — API status, DuckDB file count, latest watermark
GET /api/underlyings — List of all underlyings currently tracked in Parquet files

8.4

Pipeline Schedule Reference

Event Time (IST) Details
First live pull 09:16:30 Watermark-based incremental query. Establishes ATM window if not already saved for today.
Subsequent live pulls 09:21:30, 09:26:30 … 15:31:30 Every 5 minutes exactly, APScheduler cron trigger.
Final pull of regular session ~15:31:30 Captures 15:30 snapshot. Watermark updated to this timestamp.
Pipeline sleep 15:35 onwards Scheduler checks is_market_hours() each run; returns early if outside window.
Holiday / weekend All day is_trading_day() returns False; scheduler skips all pulls silently.
Backfill At startup (once) Runs 2026-02-17 to 2026-02-20. Skips any dates where Parquet already exists. Once done, never runs again.

8.6

Troubleshooting Guide

❌ Pipeline fails immediately on startup: "BigQuery credentials not found"
Cause: CREDENTIALS_PATH in config.py does not point to an existing JSON file, or the service account key has expired/been revoked.

Fix:
1. Confirm the key file exists at /Users/apple/Documents/OptDash/universal-ion-437606-b7-5629bd4bc421.json
2. Run: python3 -c "from google.cloud import bigquery; bigquery.Client.from_service_account_json('universal-ion-437606-b7-5629bd4bc421.json').query('SELECT 1').result()"
3. If it fails, download a fresh service account key from GCP Console → IAM → Service Accounts.
❌ BigQuery query succeeds but returns 0 rows
Cause: The watermark timestamp is in the future (timezone drift), or today is a trading holiday, or the pipeline ran outside market hours.

Debug steps:
1. cat data/watermark.json — confirm the timestamp looks like "2026-02-20 15:30:00" with no timezone suffix.
2. Check GET /api/calendar/is-trading-day?date=YYYY-MM-DD — confirm today is a trading day.
3. Manually run: python3 -c "from src.pipeline.bq_client import fetch_incremental; import pandas as pd; df=fetch_incremental('NIFTY','2026-02-20 09:00:00'); print(len(df))"
❌ Dashboard shows "API Error" or all panels are blank
Cause: FastAPI is not running, returned a non-200 status, or CORS is blocking the request.

Fix:
1. Confirm Terminal 2 shows Uvicorn running on http://127.0.0.1:8000
2. In browser, open http://127.0.0.1:8000/api/health — should return JSON.
3. If the API health endpoint returns an error, check the uvicorn terminal for Python tracebacks.
4. Confirm dashboard/.env contains VITE_API_BASE_URL=http://127.0.0.1:8000 (not localhost).
❌ DuckDB error: "No parquet files found matching glob pattern"
Cause: The date being queried has no Parquet files written (backfill hasn't completed for that date, or the pipeline hasn't run yet today).

Fix:
1. Check ls data/raw/options/ — you should see trade_date=YYYY-MM-DD/ folders.
2. If no folders exist, wait for backfill to complete. Monitor the pipeline terminal log — it should print "Completed day YYYY-MM-DD" for each of the 4 backfill dates.
3. If backfill appears stuck, check data/watermark.json. If it shows a date before 2026-02-17, the backfill hasn't started correctly. Restart run_pipeline.py.
❌ IVP values are all "N/A" or amber-coloured
Cause: Normal behaviour for the first 10 trading days. IVP requires at least 10 historical IV datapoints to produce a meaningful percentile. With only 4 days of backfill on day one, IVP cannot be calculated with statistical validity.

Solution: Continue running the pipeline through live trading sessions. IVP becomes reliable after approximately 2–3 weeks of accumulation (10 trading days minimum). The system uses amber colouring as a visual warning. Treat any IVP value displayed with fewer than 10 history datapoints as approximate.
❌ Pipeline missed a session — data gap in charts
Cause: System was stopped mid-session or an entire trading day was skipped. The pipeline records its last successful pull time in data/watermark.json.

Action: Simply restart python run_pipeline.py. The startup gap-fill engine automatically detects all scenarios:
• Mid-session restart (e.g. stopped 11:10, restarted 14:20) → fills 11:10–14:20 then starts live scheduler
• After-hours restart (stopped 11:10, restarted 17:00) → fills 11:10–15:30, exits; run again tomorrow
• Entire day skipped → pulls the full missed session automatically

No manual intervention needed — the gap fill is idempotent (safe to run multiple times).
❌ Gate score shows GO but S_score strikes are all 1-star
Cause: IVP is above 70% (options are expensive) and/or bid-ask spreads are very wide (low liquidity period — early morning or after a large move).

Action: Despite a GO Gate, do NOT enter if no 3+ star strikes exist. Wait 1–2 snapshots for spreads to normalise. If IVP remains above 70%, this is a NO-TRADE day for directional option buying regardless of Gate score.
❌ run_pipeline.py crashes with "ModuleNotFoundError: No module named 'src'"
Cause: Running python run_pipeline.py from the wrong working directory, or the virtual environment is not activated.

Fix:
1. Always run from the project root: cd /Users/apple/Documents/OptDash
2. Activate venv first: source .venv/bin/activate
3. Confirm you see (.venv) in your prompt before running any Python command.
❌ Vite dev server fails: "Cannot find module '@tanstack/react-query'"
Cause: Node modules not installed or partially installed.

Fix: cd /Users/apple/Documents/OptDash/dashboard && npm install. If still failing: rm -rf node_modules && npm install.
8.7

Glossary of Terms

ATM (At-the-Money)
An option whose strike price equals the current spot price. Has the highest gamma and maximum extrinsic value.
ATM Window
The ±8 strike range computed from the day-open spot price, fixed for the entire session. Only strikes within this window appear in the Screener.
BigQuery
Google's serverless cloud data warehouse providing the raw NSE F&O tick data. Queried via the Python client using the service account key.
CoC (Cost of Carry)
The futures premium above spot: CoC = Futures LTP – Spot. Theoretically driven by the risk-free rate minus dividends, adjusted for DTE.
CE (Call European)
The NSE notation for a call option on an Indian index. Gives the right but not obligation to buy at the strike price.
Delta (Δ)
Option price change per ₹1 move in the underlying. Ranges 0–1 for calls, −1 to 0 for puts. Approximately 0.5 at ATM.
DTE (Days To Expiry)
Calendar days remaining until option expiry date. Lower DTE = higher theta cost, higher gamma sensitivity.
DuckDB
In-process analytical SQL engine used to query Parquet files. No server needed — runs embedded in the Python process.
EffRatio
Efficiency Ratio: |Delta| / |Theta|. Higher is better for option buyers — more directional sensitivity per unit of daily time decay.
Expiry Tier
Classification of options by DTE: TIER1_NEAR (1–12 DTE), TIER1_FAR (13–34 DTE), TIER2_IV_ONLY (35+ DTE).
FastAPI
Python async web framework serving the analytics API at localhost:8000. Serves DuckDB query results as JSON to the React dashboard.
FII (Foreign Institutional Investor)
Large foreign fund houses trading Indian markets. Their futures activity is the primary source of directional institutional flow detected by V_CoC.
Gamma (Γ)
Rate of change of delta per ₹1 spot move. Long option buyers have positive gamma — their delta grows favourably as spot moves toward them.
Gate (Environment Gate)
8-condition scoring system (0–11 pts; 6 core + 2 VEX/CEX bonus). GO = ≥5, WAIT = 3–4, NO-GO <3. Must be GO before any option entry.
GEX (Gamma Exposure)
Net gamma position of all market makers expressed in ₹ notional terms. High GEX suppresses directional moves; low/declining GEX enables trending.
IST (India Standard Time)
UTC+5:30. All timestamps in the system are naive IST — no timezone conversion is ever applied.
ITM (In-the-Money)
For a call: strike below spot. For a put: strike above spot. ITM options have intrinsic value in addition to time value.
IV (Implied Volatility)
The market's consensus annualised volatility forecast, reverse-engineered from option prices via Black-Scholes.
IV Crush
A rapid drop in IV after a scheduled event, destroying long option value even when the directional move was correct.
IVP (IV Percentile)
Fraction of the last 90 trading days where IV was below today's IV. Below 50% = cheap options. Preferred metric over IVR.
IVR (IV Rank)
Position of today's IV within the 90-day high-low range. Can be distorted by outlier spikes. Use IVP instead.
OBI (Order Book Imbalance)
Metric from −1 to +1 measuring the ratio of visible buy to sell quantities in the top-5 limit order book levels.
OI (Open Interest)
Total number of outstanding contracts (not yet settled). Rising OI = new positions being built. Falling OI = positions being unwound.
OTM (Out-of-the-Money)
For a call: strike above spot. For a put: strike below spot. OTM options consist entirely of time value.
Parquet
Columnar binary file format used for storing NSE data. Snappy-compressed, row_group_size=100,000. Enables fast DuckDB queries via file pruning.
PCR (Put-Call Ratio)
Ratio of put volume/OI to call volume/OI. Higher = more put activity. The dashboard tracks both Volume-PCR and OI-PCR separately.
PCR Divergence
|PCR_vol − PCR_oi|. Values above 0.25 suggest retail sentiment has diverged from institutional structure (Gate condition #4).
PE (Put European)
NSE notation for a put option. Gives the right to sell at the strike price. Profitable when underlying falls below strike − premium paid.
S_score
Strike Selection Score (0–100). Composite of EffRatio, slippage penalty, IVP adjustment, and OBI alignment. Higher = better strike to buy.
SL (Stop Loss)
The price level at which a losing position is closed to limit further losses. The dashboard uses a theta-adjusted dynamic SL that decreases slowly as time passes.
Theta (Θ)
Daily rate of option price decay from time passing. Always negative for long option buyers. Expressed as ₹ lost per calendar day.
V_CoC (Velocity of CoC)
15-minute rate of change of futures carry. The direction detection signal for institutional flow. Below −10 = bearish; above +10 = bullish.
Vega (ν)
Option price change per 1% change in implied volatility. Long option buyers benefit from rising IV (positive vega P&L).
Vite
Modern frontend build tool and development server. Serves the React TypeScript dashboard at localhost:5173 with hot module replacement.
Volatility Skew
The observation that OTM puts consistently have higher IV than equivalent OTM calls. Caused by institutional tail-risk hedging demand.
VEX (Vanna Exposure)
Dealer hedging flow driven by IV changes. When IV drops, dealers holding short puts/calls must rebalance delta by buying/selling futures. Positive VEX = mechanical buying pressure on NIFTY. Computed as OI × vanna × spot × sigma × d_dir / 1e6 (M-units).
CEX (Charm Exposure)
Dealer hedging flow driven by time-decay. As OTM options approach expiry, their delta decays toward zero — dealers buy back futures accordingly. Largest on expiry day at 14:45–15:25 IST ("Dealer O'Clock"). Computed from charm_daily × OI × d_dir / 1e6 (M-units).
Dealer O'Clock
14:45–15:25 IST on expiry day (minutes 885–925). Window when charm flows from expiring options dominate index movement. CEX panel shows orange ⏰ badge. Gate condition #8 suppresses option-buyer entries during this window.
Watermark
The timestamp of the most recent successful BigQuery pull, stored in data/watermark.json. Prevents re-fetching data that has already been ingested.
